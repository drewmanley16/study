Q6 — Greedy: Announcement Scheduling 15 pts ▼
n workers each have a shift interval [sᵢ, fᵢ]. Make as few announcements as possible so every worker hears at least one (a worker hears an announcement at time t if sᵢ ≤ t ≤ fᵢ).

Q6.1 (7 pts) Design a greedy algorithm.

Q6.2 (8 pts) Prove its correctness.
Solution
**Q6.1 Greedy Algorithm:**
1. Sort workers' shifts by **increasing finish time** f₁ ≤ f₂ ≤ ... ≤ fₙ.
2. Initialize announcement set A = {}.
3. While uncovered workers remain:
- Take the uncovered worker with the earliest finish time fᵢ.
- Add fᵢ to A (announce at the last possible moment of that shift).
- Remove all workers whose shift contains fᵢ.
4. Return A.

Intuition: announcing at the latest moment of the earliest-ending shift covers the most workers.

**Q6.2 Proof of Correctness (exchange argument):**
Sort so f₁ ≤ f₂ ≤ ... ≤ fₙ. Let greedy picks g₁ < g₂ < ... < gᵣ; optimal picks c₁ < c₂ < ... < cₖ with k minimal.

**Claim:** gᵢ ≥ cᵢ for all i (greedy announcements are "at least as late").
- Base: g₁ = f₁ (earliest finish time). Optimal must cover worker 1, so s₁ ≤ c₁ ≤ f₁, meaning g₁ ≥ c₁.
- Inductive step: assume gᵢ ≥ cᵢ. Since gᵢ is at least as late as cᵢ, greedy covers ≥ as many workers. So the next uncovered worker for greedy has finish time fₘ₊₁ ≥ fₘ′₊₁, giving gᵢ₊₁ = fₘ₊₁ ≥ fₘ′₊₁ ≥ cᵢ₊₁.

Since {g₁,...,gₖ} covers all workers, r ≤ k, contradicting r > k. So greedy is optimal.
Graphs: BFS, DFS, SCC, Topological Sort▼
BFS (Breadth-First Search):
- Uses a queue; explores level by level
- Finds shortest path in unweighted graphs
- Time: O(V+E)

DFS (Depth-First Search):
- Uses a stack (or recursion); explores as deep as possible first
- Used for cycle detection, topological sort, SCC
- Time: O(V+E)
- Discovery time d[v] and finish time f[v] are key

Topological Sort:
- Order vertices so all edges go from earlier to later
- Only on DAGs (directed acyclic graphs)
- Algorithm: run DFS, output vertices in decreasing finish time order

Strongly Connected Components (SCC):
- Kosaraju-Sharir: (1) DFS on G, record finish order; (2) DFS on Gᵀ in reverse finish order
- Each DFS tree in step 2 = one SCC
- Time: O(V+E)
Minimum Spanning Trees: Kruskal & Prim▼
MST: spanning tree of G with minimum total edge weight.

Kruskal's Algorithm:
- Sort all edges by weight
- Add edge if it doesn't form a cycle (use Union-Find)
- Greedy: always picks the globally cheapest safe edge
- Time: O(E log E)

Prim's Algorithm:
- Start from any vertex, greedily expand the tree
- Always add the cheapest edge connecting tree to non-tree vertex
- Use a min-heap for efficiency
- Time: O(E log V) with binary heap

Key properties:
- Cut property: the minimum weight edge crossing any cut is in every MST
- Cycle property: the maximum weight edge in any cycle is NOT in any MST
Shortest Paths: Dijkstra, Bellman-Ford▼
Dijkstra's Algorithm (non-negative weights):
- Greedy: always relax the vertex with smallest known distance
- Use a min-priority queue
- Time: O((V+E) log V) with binary heap
- Does NOT work with negative edge weights

Bellman-Ford (negative weights OK):
- Relax all edges V-1 times
- If you can still relax after V-1 iterations → negative cycle detected
- Time: O(VE)
- Works on graphs with negative edges (but not negative cycles)

Floyd-Warshall (all-pairs shortest path):
- DP: d[i][j][k] = shortest path from i to j using only vertices {1,...,k}
- Recurrence: d[i][j][k] = min(d[i][j][k-1], d[i][k][k-1]+d[k][j][k-1])
- Time: O(V³)

Acyclic Shortest Path:
- On a DAG: relax edges in topological order
- Time: O(V+E), works with negative edges
Dynamic Programming▼
DP Template:
1. Define sub-problems
2. Write recurrence relation
3. Identify base cases
4. Compute bottom-up (or memoize top-down)
5. Reconstruct solution

Matrix Chain Multiplication:
- M(i,j) = min over k: M(i,k)+M(k+1,j)+d_{i-1}·dₖ·dⱼ
- Fill table by increasing chain length

Longest Common Subsequence (LCS):
- dp[i][j] = LCS of X[1..i] and Y[1..j]
- If X[i]==Y[j]: dp[i][j] = dp[i-1][j-1]+1
- Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])

Longest Increasing Subsequence (LIS):
- dp[i] = length of LIS ending at index i
- O(n²) DP; O(n log n) with patience sorting

0/1 Knapsack:
- dp[i][w] = max value using items 1..i with capacity w
- dp[i][w] = max(dp[i-1][w], vᵢ + dp[i-1][w-wᵢ]) if wᵢ ≤ w

Bellman-Ford as DP:
- dp[k][v] = shortest path from s to v using at most k edges
- Recurrence: dp[k][v] = min over (u,v): dp[k-1][u] + w(u,v)
Greedy Algorithms▼
Greedy: make locally optimal choice at each step; prove globally optimal.

Proof techniques:
- Exchange argument: show swapping any greedy choice with optimal doesn't improve
- "Greedy stays ahead": show greedy is at least as good as optimal at each step

Activity Selection (Interval Scheduling):
- Sort by finish time; always pick the next compatible activity
- Greedy choice: earliest finish time
- Correctness: exchange argument

Huffman Coding:
- Greedy: always merge two lowest-frequency nodes
- Produces optimal prefix-free code (minimum weighted path length)
- Proof: if two symbols have lowest frequency, they can have the longest codes

Fractional Knapsack:
- Sort items by value/weight ratio; fill greedily
- NOT the same as 0/1 knapsack (which requires DP)
Max Flow & Min Cut (Ford-Fulkerson)▼
Definitions:
- Flow network: directed graph with capacities c(u,v)
- Valid flow: 0 ≤ f(u,v) ≤ c(u,v); flow conservation at all non-source/sink nodes
- Max flow: maximum total flow from s to t

Ford-Fulkerson Algorithm:
1. Start with 0 flow
2. While augmenting path exists in residual graph:
- Send flow along that path (bottleneck = min residual capacity)
- Update residual graph
3. Residual graph: for edge (u,v) with capacity c and flow f:
- Forward edge: c-f remaining capacity
- Backward edge: f (can "undo" flow)

Time: O(E · |max flow|) — can be slow with irrational capacities; Edmonds-Karp (BFS) = O(VE²)

Max-Flow Min-Cut Theorem:
max flow = min cut capacity
- Min cut: partition (S,T) with s∈S, t∈T minimizing sum of capacities of edges from S to T
- The algorithm finds the min cut implicitly: S = vertices reachable from s in residual graph when algorithm terminates
Divide & Conquer▼
Pattern:
1. Divide problem into subproblems
2. Conquer subproblems recursively
3. Combine solutions

Master Theorem (T(n) = aT(n/b) + f(n)):
- If f(n) = O(n^{log_b(a) - ε}): T(n) = Θ(n^{log_b a})
- If f(n) = Θ(n^{log_b a}): T(n) = Θ(n^{log_b a} · log n)
- If f(n) = Ω(n^{log_b(a) + ε}) and af(n/b) ≤ cf(n): T(n) = Θ(f(n))

Karatsuba Integer Multiplication:
- Standard: O(n²); Karatsuba: O(n^{log₂3}) ≈ O(n^{1.585})
- Split n-bit numbers: x = x_H·2^{n/2} + x_L
- Compute 3 multiplications instead of 4: (x_H+x_L)(y_H+y_L), x_H·y_H, x_L·y_L
- T(n) = 3T(n/2) + O(n) → O(n^{log₂3})

Merge Sort: T(n) = 2T(n/2) + O(n) → O(n log n)

Question
What is the time complexity of BFS and DFS on a graph with V vertices and E edges?
Answer
Both BFS and DFS run in Θ(V + E) time when using an adjacency-list representation.
Question
What is the key difference between DFS preorder and postorder?
Answer
Preorder records a vertex when DFS is first called on it; postorder records a vertex when DFS returns from it. Reverse postorder gives a topological ordering of a DAG.
Question
State the correctness condition for topological sort via DFS.
Answer
Reverse DFS postorder of a DAG is a valid topological order. For any edge v→w, DFS(w) finishes before DFS(v) finishes, so w appears after v in reverse postorder.
Question
When does a digraph have a topological ordering?
Answer
A digraph has a topological ordering if and only if it contains no directed cycle (i.e., it is a DAG).
Question
State the Cut Property for Minimum Spanning Trees.
Answer
For any cut of a connected graph, the minimum-weight crossing edge is guaranteed to be in the MST.
Question
What is the time complexity of Kruskal's MST algorithm?
Answer
O(E log E) due to sorting edges; union-find operations are nearly linear, so the bottleneck is sorting.
Question
What is the time complexity of lazy Prim's MST algorithm?
Answer
Θ(E log E) time and Θ(E) extra space, since each edge is inserted and deleted from the priority queue at most once.
Question
What is the key difference between Prim's algorithm and Dijkstra's algorithm?
Answer
Prim selects the minimum-weight edge connecting any non-tree vertex to the tree; Dijkstra selects the vertex with the minimum cumulative distance from the source. Both use a priority queue but optimize different objectives.
Question
State the edge relaxation condition used in Dijkstra's and Bellman-Ford algorithms.
Answer
For edge e = v→w with weight e.weight: if distTo[w] > distTo[v] + e.weight, then update distTo[w] = distTo[v] + e.weight and edgeTo[w] = e.
Question
What is the time complexity of Dijkstra's algorithm using a binary heap priority queue?
Answer
O(E log V), since each vertex is extracted once (O(V log V)) and each edge relaxation costs O(log V), giving O(E log V) total.
Question
What is the Bellman-Ford algorithm and when is it preferred over Dijkstra?
Answer
Bellman-Ford relaxes all E edges V times, running in O(VE) time. It is preferred when edge weights can be negative (but no negative cycles exist), where Dijkstra is not applicable.
Question
What are the two key elements required for a dynamic programming solution?
Answer
Optimal substructure (optimal solutions to sub-problems combine to form the optimal solution to the whole problem) and overlapping subproblems (the same sub-problems recur multiple times).
Question
Write the DP recurrence for the Longest Increasing Subsequence (LIS) using the formulation L[i,j].
Answer
L[i,j] = 0 if i<0; 𝕀(A[i]≤A[j]) if i=0; L[i−1,j] if A[i]>A[j]; max{1+L[i−1,i], L[i−1,j]} otherwise. The overall LIS length is max_j L[n−1,j]. Runtime is O(n²).
Question
What is the recurrence for the 0/1 Knapsack problem?
Answer
Let OPT(i,w) = max value using items 1..i with capacity w. Then OPT(i,w) = OPT(i−1,w) if item i does not fit (w_i > w), else max{OPT(i−1,w), v_i + OPT(i−1,w−w_i)}. Runtime is O(nW).
Question
What is the optimal substructure property used in the rod-cutting problem?
Answer
The maximum revenue r_n = max over i of {p_i + r_{n−i}}, meaning an optimal solution for a rod of length n incorporates optimal solutions for the two sub-rods created by the first cut.
Question
What is the time complexity of the bottom-up rod-cutting DP algorithm?
Answer
Θ(n²), because for each length j from 1 to n, we iterate over all i from 1 to j, giving a total of n(n+1)/2 operations.
Question
What is Huffman coding and what greedy property does it use?
Answer
Huffman coding builds an optimal prefix-free binary code by greedily merging the two lowest-frequency symbols into a single node, repeated until one tree remains. It is optimal because the two rarest symbols have the longest codewords and can always be placed at the deepest level.
Question
What is the Fibonacci DP recurrence and why is the naive recursive approach exponential?
Answer
F(n) = F(n−1) + F(n−2) with F(1)=F(2)=1. The naive approach is exponential (Ω(2^{n/2})) because it recomputes the same sub-problems repeatedly; memoization or bottom-up DP reduces this to O(n).
Question
What is the time complexity of the Floyd-Warshall all-pairs shortest paths algorithm and what DP substructure does it use?
Answer
Floyd-Warshall runs in O(V³) time. Its substructure: the shortest path from i to j using only intermediate vertices from {1,...,k} is either the path not using k, or the path through k, giving d^(k)[i][j] = min{d^(k−1)[i][j], d^(k−1)[i][k] + d^(k−1)[k][j]}.
Question
What is the independent set problem on a path graph and how is it solved with DP?
Answer
Given a path graph with weighted vertices, find a maximum-weight subset of vertices with no two adjacent. The recurrence is OPT(i) = max{OPT(i−1), w_i + OPT(i−2)}, solved in O(n) time by building up from the base cases OPT(0)=0 and OPT(1)=w_1.

Q2 — Rotate 2D Array (C) 15 pts ▼
Implement `void rotate_arr(int mat, int m)` that rotates an m×m matrix 90° counter-clockwise in-place (storing result back in mat). You may use a temporary array. No memory leaks!

Example: [[1,2,3],[4,5,6],[7,8,9]] → [[3,6,9],[2,5,8],[1,4,7]]
Solution
```c
void rotate_arr(int** mat, int m) {
// Allocate temp
int** tmp = malloc(sizeof(int*) * m);
for(int i = 0; i < m; i++)
tmp[i] = malloc(sizeof(int) * m);

// Copy with rotation: CCW means new[m-1-j][i] = old[i][j]
for(int i = 0; i < m; i++)
for(int j = 0; j < m; j++)
tmp[m-1-j][i] = mat[i][j];

// Copy back
for(int i = 0; i < m; i++)
for(int j = 0; j < m; j++)
mat[i][j] = tmp[i][j];

// Free temp
for(int i = 0; i < m; i++) free(tmp[i]);
free(tmp);
}
```
**In-place (extra credit):** transpose then flip columns:

```c
// Transpose
for(i) for(j>i) swap(mat[i][j], mat[j][i]);
// Flip columns (reverse each column)
for(j) for(i < m/2) swap(mat[i][j], mat[m-1-i][j]);
```
Q3 — Prefix Sum In-Place (C) 6 pts ▼
Implement `void prefix_sum(int* A, int m)` that computes prefix sum in-place (no extra storage, no redundant operations).

Recall: y[i] = x[0] + x[1] + ... + x[i]
Solution
```c
void prefix_sum(int* A, int m) {
for(unsigned int i = 1; i < m; i++) {
A[i] = A[i] + A[i-1];
}
}
```
Key points:
- Start loop at i=1 (A[0] is already its own prefix sum)
- No temporary variable needed — A[i-1] is already the prefix sum up to i-1 when we reach index i
- Do NOT start at i=0 (that would be a redundant operation: A[0] = A[0] + ... nothing)

Q5 — Destructor & Copy Constructor (C++) 8 pts ▼
Given:
```cpp
class Test {
int* array;
int n;
// ...
};
void f(Test t) { delete[] t.array; t.n = 0; }
```
(a) (2 pts) Declare the destructor and copy constructor.
(b) (2 pts) Implement the destructor.
(c) (4 pts) Implement the copy constructor so that passing t1 to f() does NOT destroy t1's array.
Solution
**(a) Declarations:**
```cpp
~Test();
Test(const Test& in);
```

**(b) Destructor:**
```cpp
Test::~Test() {
delete[] array;
array = nullptr; // optional but good practice
n = 0; // optional
}
```

**(c) Copy Constructor (deep copy):**
```cpp
Test::Test(const Test& in) {
n = in.n;
array = new int[n];
for(unsigned int i = 0; i < n; i++)
array[i] = in.array[i];
}
```
WHY: Without a copy constructor, C++ does a **shallow copy** (copies the pointer, not the data). When f() deletes t.array, it deletes the same memory t1.array points to. Deep copy allocates new memory, so f() deletes its own copy and t1 is unaffected.

C: Pointers, Memory, and Arrays▼
Pointers:
- int* p = &x; → p holds address of x
- *p → dereference (get value at address)
- p++ → move to next int-sized address
- Arrays decay to pointer: int A[] == int* A

Pointer arithmetic:
- A[i] == *(A+i) always equivalent
- For 1-indexed: A[i-1] or *(A+i-1)

Heap memory (manual):
- malloc(n * sizeof(T)) → allocate n elements of type T
- free(ptr) → release heap memory
- MUST free everything malloc'd → no memory leaks
- After free: set ptr = NULL to avoid dangling pointer

Common memory pitfalls:
- Memory leak: allocate but never free
- Double free: free the same pointer twice
- Use after free: access memory after freeing
- Buffer overflow: write past end of array

sizeof:
- sizeof(struct) may be padded for alignment
- sizeof(int*) = 8 on 64-bit, sizeof(int) = 4 typically

2D arrays:
- int** A = malloc(sizeof(int*) * rows);
- for each row: A[i] = malloc(sizeof(int) * cols);
- Free: free each row, then free A
C++: OOP, Constructors, Destructors▼
Constructor/Destructor order:
- Base class constructor runs BEFORE derived class constructor
- Destructor order is REVERSED: derived first, then base
- Always declare destructor virtual if using polymorphism

Copy Constructor:
- Called when passing by value, returning by value, or explicit copy
- Default (shallow) copy: copies pointers — shared memory!
- Deep copy: allocate new memory and copy contents
- Signature: ClassName(const ClassName& other)

Rule of Three:
If you need a custom destructor, you also need a custom copy constructor AND copy assignment operator.

Access control:
- private: only accessible within the class
- protected: accessible in class + derived classes
- public: accessible everywhere

const member functions:
- Cannot modify any member variables
- Can be called on const objects
- Declared with const after parameter list: void foo() const;

Static members:
- Shared across ALL instances of the class
- Static data: one copy for the whole class
- Static method: no 'this' pointer, can only access static members

BST & Red-Black Trees▼
BST (Binary Search Tree):
- left subtree keys < node key < right subtree keys
- Search/Insert/Delete: O(h) where h = height
- Worst case: O(n) if unbalanced (sorted insertions)
- In-order traversal gives sorted order

Red-Black Tree (balanced BST):
Properties (every RBT must satisfy all 5):
1. Every node is red or black
2. Root is black
3. Every leaf (NIL) is black
4. If a node is red, both children are black (no two consecutive reds)
5. All paths from a node to its NIL descendants have the same number of black nodes

Height: O(log n) — guaranteed by these properties

Insert:
- Insert as in BST, color new node RED
- Fix violations: recolor and/or rotate
- Cases: uncle red (recolor), uncle black with two cases (rotations)

Delete:
- Remove as in BST
- If deleted node was black: "double black" violation → fix with rotations/recoloring
- Cases for fix-up: sibling red, sibling black with various child colors

Rotations:
- Left rotation: x's right child y becomes new root; x becomes y's left child
- Right rotation: symmetric
- O(1) pointer changes

Question
What operator is used to get the memory address of a variable in C?
Answer
The & (address-of) operator. Example: int* p = &x; stores the address of x in p.
Question
What is pointer arithmetic and how does it scale?
Answer
Arithmetic on pointers scales by the size of the pointed-to type. e.g., if int* p = 0x8000, then p++ gives 0x8004 (4 bytes for int); double* d = 0x6000, d++ gives 0x6008 (8 bytes for double).
Question
What does malloc() do and what must always be done after using it?
Answer
malloc() allocates a specified number of bytes on the heap and returns a pointer to the block. After use, free() must be called on the pointer to prevent memory leaks.
Question
What are the four memory segments in a C program?
Answer
Text (code), Data (global/static variables), Stack (static/local memory, short-lived), and Heap (dynamic memory, lives until freed or program ends).
Question
What is the difference between stack and heap memory allocation?
Answer
Stack memory is automatically managed, short-lived (lasts during a function call), and fast. Heap memory is manually managed with malloc/free, long-lived, and more flexible but slower to allocate.
Question
What is a shallow copy of a struct and why can it cause problems?
Answer
A shallow copy copies each member value individually. If a member is a pointer, only the address is copied (not the pointed-to data), so both structs point to the same memory, causing issues if one frees it.
Question
What are the three access control types in C++ and what do they mean?
Answer
public: accessible by anything; private: accessible only inside the class; protected: accessible inside the class and inside derived (inherited) classes.
Question
How do structs and classes differ in terms of default access control in C++?
Answer
In a struct, members are public by default. In a class, members are private by default. Otherwise they are nearly identical.
Question
What is a constructor in C++ and what are its key characteristics?
Answer
A constructor initializes class members automatically when an object is created. It has the same name as the class, no return type, cannot be called explicitly, and runs once per object.
Question
What is constructor overloading in C++?
Answer
Defining multiple constructors for a class with different parameter lists, allowing objects to be initialized in different ways. e.g., Rectangle(int w, int h) and Rectangle(int d).
Question
What is the BST property?
Answer
For any node x in a BST: all keys in the left subtree are <= key[x], and all keys in the right subtree are > key[x].
Question
What is the time complexity of BST operations and what determines it?
Answer
O(log n) if the tree is balanced/complete; O(n) if the tree degenerates into a linear chain. The height of the tree determines complexity.
Question
How do you find the minimum and maximum keys in a BST?
Answer
Minimum: follow left child pointers until reaching a node with no left child. Maximum: follow right child pointers until reaching a node with no right child.
Question
What are the three cases for BST deletion?
Answer
Case 1: Node is a leaf – set parent's pointer to NIL. Case 2: Node has one child – replace node with its child. Case 3: Node has two children – replace node with its in-order successor, then splice out the successor.
Question
What are the 5 Red-Black Tree properties?
Answer
1) Every node is RED or BLACK. 2) Root is BLACK. 3) Every leaf (NIL/sentinel) is BLACK. 4) If a node is RED, both children are BLACK. 5) All paths from any node to descendant leaves have the same number of BLACK nodes.
Question
What is a sentinel in a Red-Black Tree and why is it used?
Answer
A sentinel is a single BLACK Node* that replaces all NIL/nullptr pointers in the RBT. It simplifies implementation by ensuring all leaf and missing-child pointers point to a real (though dummy) node rather than null.
Question
What are rotations in a Red-Black Tree and why are they needed?
Answer
Left and right rotations restructure the tree locally to maintain or restore RBT properties after insertions or deletions that may violate them, while preserving the BST property.
Question
What steps are taken when inserting a node into a Red-Black Tree?
Answer
1) Insert the node using standard BST insertion. 2) Color the new node RED. 3) Call rb-insert-fixup to perform recoloring and rotations to restore RBT properties.
Question
What Unix compile flags are required for C and C++ code in CS330?
Answer
C code must compile with -std=c11; C++ code must compile with -std=c++14. Code that does not compile receives a 0.
Question
What is the difference between static and dynamic libraries?
Answer
Static libraries (created with ar) include all required function code directly in the executable, making it larger but self-contained. Dynamic libraries (created with gcc --shared) are linked at runtime, resulting in a smaller executable but requiring the library at runtime.

A low discount rate (1%) makes that same $1 worth $0.61 today → SCC is much higher (~$200). Even small differences in r compound dramatically over long time horizons. (g) CO₂ mixes globally — one country's emissions affect all countries equally. This creates a global public goods problem: each country free-rides on others' reductions. Local pollution (e.g., city smog) is contained and affects the emitter → easier to regulate nationally. International attempts: Paris Agreement (voluntary NDCs), EU ETS (cap-and-trade), proposed carbon border adjustment mechanisms (CBAMs).
1. Normative vs. Positive Analysis▼
POSITIVE analysis: describes what IS — empirical, testable claims about how the world works. Examples: "A carbon tax reduces emissions." "Wetland loss decreases water quality." NORMATIVE analysis: describes what OUGHT to be — value judgments about desirable outcomes. Examples: "We should reduce CO₂ by 50%." "The government should subsidize clean energy." Economics uses positive analysis to evaluate tradeoffs; normative choices determine the policy goals. Know which type of claim you're making — exams may ask you to identify statements as normative or positive.
2. MB & MC Curves — Individual and Aggregate▼
PRIVATE GOODS — Aggregate demand HORIZONTALLY (sum quantities at each price): Qₐᵍᵍ(P) = Q₁(P) + Q₂(P) + ... + Qₙ(P) Example: Tim P=5−Q → Q=5−P; Jin P=5−0.5Q → Q=10−2P Qₐᵍᵍ = (5−P) + (10−2P) = 15−3P PUBLIC GOODS — Aggregate MB VERTICALLY (sum WTP at each quantity): SMB(Q) = MB₁(Q) + MB₂(Q) + ... + MBₙ(Q) Example: Tim MB=5−Q; Jin MB=5−0.5Q SMB = (5−Q) + (5−0.5Q) = 10−1.5Q ABATEMENT MC CURVES — Aggregate HORIZONTALLY (same logic as private good demand): Invert each MC, sum quantities, re-invert to get MC_agg. Example: MC₁=10A₁ → A₁=MC/10; MC₂=5A₂ → A₂=MC/5 Aₐᵍᵍ = MC/10 + MC/5 = 3MC/10 → MC_agg = (10/3)A_agg KEY DISTINCTION: Horizontally for individual choices (private goods, MAC curves). Vertically for collective WTP (public goods, externality benefits).
3. Public vs. Private Goods + Samuelson + Lindahl▼
TAXONOMY: Rival + Excludable = Private Good (food, clothing) Non-rival + Excludable = Club Good (Netflix) Rival + Non-excludable = Common Resource (fisheries) Non-rival + Non-excludable = Public Good (clean air, national defense) PUBLIC GOOD PROBLEM: Non-excludability → free-rider problem → markets underprovide. SAMUELSON CONDITION — Efficient level of public good: Σᵢ MBᵢ(Q*) = MC(Q*) i.e., SMB = MC Intuition: since all individuals consume the same Q, the total benefit of one more unit is the sum of everyone's MB, not just one person's. LINDAHL SOLUTION — each person pays their own MB at Q* as a personalized "price": Lindahl price for person i = MBᵢ(Q*) Sum of Lindahl prices = MC(Q*) ✓ Problem: Requires knowing everyone's true preferences → not feasible in practice. VOLUNTARY CONTRIBUTION: Each person chooses Q to max own utility, taking others' contributions as given → Nash equilibrium has severe underprovision relative to Samuelson optimum. VOTING: Majority vote → median voter's preferred Q. Often below Q* because voters don't internalize others' MB.
4. Welfare: TB, TC, NB, MB, MC, DWL, Efficiency▼
KEY RELATIONSHIPS: MB = dTB/dA (marginal = derivative of total) MC = dTC/dA Net Benefit (NB) = TB − TC Maximized when MB = MC (set derivative of NB to zero) FINDING EFFICIENT LEVEL (A*): 1. Take derivatives: MB = dTB/dA, MC = dTC/dA 2. Set MB = MC and solve for A* 3. NB* = TB(A*) − TC(A*) = ∫₀^A* [MB(A) − MC(A)] dA MARKET EQUILIBRIUM vs. EFFICIENT EQUILIBRIUM: Market: PMB = PMC (ignores externalities) Efficient: SMB = SMC (includes all social costs/benefits) EXTERNALITY TAXONOMY: Negative production: SMC > PMC → overproduction (market Q > efficient Q) Negative consumption: SMB < PMB → overproduction Positive production: SMC < PMC → underproduction Positive consumption: SMB > PMB → underproduction DWL (Deadweight Loss): Negative externality DWL = ∫_{Q*}^{Q_m} (SMC − SMB) dQ (triangle area) Positive externality DWL = ∫_{Q_m}^{Q*} (SMB − SMC) dQ To draw: find where MB=SMC (efficient pt) and where PMB=PMC (market pt); DWL is the triangle between these points. EFFICIENCY vs. COST-EFFECTIVENESS: Efficient: MB = MC → maximizes NB (best possible outcome) Cost-effective: MC₁ = MC₂ = ... = MC_j for a given total abatement target → minimizes TAC Efficient → always cost-effective. Cost-effective → NOT necessarily efficient (target might be wrong level).
5. Policy Solutions to Pollution — All 8 Types▼
POLICY I — Direct Abatement Subsidy: Gov't pays firm P* per unit of abatement. Firm abates until MC = P*. Efficient if P* set at MB(A*). Cost-effective: yes (all firms face same price P*). Total subsidy = P* × A*. Real examples: EPA Brownfield grants, $7,500 EV credit. POLICY II — Pigouvian (Indirect) Tax: Tax on the market GOOD (not the pollutant). Raises effective MC → internalizes externality. Tax = P_D − P_S at efficient Q* (wedge between buyer and seller price). Set where SMB = SMC. Revenue = tax × Q*. Double Dividend: (1) corrects the externality market, AND (2) revenue can reduce distortionary taxes (income tax) → moves THAT market closer to efficiency too. POLICY III — Coase Theorem (Market Solution): If (1) property rights are clear and (2) zero transaction costs → private bargaining achieves efficient outcome regardless of who has the property right. Distributional impact differs by who holds rights. Fails in practice: high transaction costs, many parties, undefined rights. POLICY IV — Direct Tax on Pollution: Gov't sets tax P* per unit of EMISSIONS (not the market good). Firm pays tax on pollution emitted. Total pollution = A_Total − A* (what's NOT abated). Tax revenue = P* × (A_Total − A*). Firm abates until MC = tax (same as abatement subsidy mechanism). Efficient if set at P* = MB(A*). Cost-effective: yes. Real examples: SF Bay Area carbon tax (4.4¢/ton CO₂), WI/CA/LA effluent taxes. POLICY V — Uniform Quantity Standard (UQS / Command & Control): Each firm must abate the same fixed amount. Firm abates regardless of their MC. Does NOT equate MCs across firms → NOT cost-effective (unless all firms have identical MACs). Not efficient either. But it does achieve the abatement target. TAC_UQS = Σᵢ TCᵢ(A/n) — each firm abates A/n. Political economy: firms prefer UQS over taxes (avoid paying tax on residual pollution); lobbying explains why UQS is common despite economic inefficiency. Real examples: NAAQS (Clean Air Act), Total Maximum Daily Loads for water. POLICY VI — Cap and Trade (Emissions Trading): Gov't sets total cap on emissions, distributes permits, allows trading. Firms trade until MAC equalized at permit price P*. Cost-effective: yes. Efficient: only if cap set at A*. Key mechanic: firm abates if MAC < permit price; buys permits if MAC > permit price. Hot spots: only a concern for FLOW (local) pollutants. For global stock pollutants (CO₂), location of emission doesn't matter → trading is fine. Real: Acid Rain Program (SO₂), EU ETS (CO₂). POLICY VII — Technology Standard: Regulator requires a minimum technology standard for production (e.g., best available control technology). Not efficient (can't know the right technology for all firms). Not cost-effective (doesn't equate MACs). Advantage: certainty that technology will be adopted. Disadvantage: no incentive to innovate beyond minimum. POLICY VIII — Weitzman (1974): Prices vs. Quantities Under Uncertainty: Under uncertainty about MC curve: → Steep MB relative to MC: prefer QUANTITY instrument (cap). Getting the quantity wrong is costly. → Flat MB relative to MC: prefer PRICE instrument (tax). Getting the price wrong is less costly. For climate: MB of abatement is quite flat → prefer carbon TAX over cap-and-trade. COMPARISON TABLE: Policy | Efficient? | Cost-Effective? 6. Types of Value & Non-Market Valuation Overview▼
WHY VALUATION? Environmental goods (clean air, biodiversity, recreation) don't have market prices. We need dollar values for cost-benefit analysis and policy design. TAXONOMY OF VALUE (additive & separable — don't double count): USE VALUE: Direct: utility from direct use/engagement (hiking, fishing, swimming) Indirect: value as an input to something valuable (wetlands filter water → lower water treatment cost) NON-USE VALUE: Bequest value: WTP knowing that future generations will have access to the good Option value: WTP to preserve the OPTION to use something in the future (separate from actual use value) Existence value: utility derived simply from knowing something exists (e.g., knowing whales exist, even if you'll never see one) KEY METRIC — WTP vs. WTA: WTP: max amount a person would pay for more of a good (or less of a bad). Bounded by income. WTA: min compensation for accepting more of a bad (or less of a good). WTA ≥ WTP in practice. Why? Income effects, loss aversion (prospect theory), ownership effects. MARKET vs. NON-MARKET VALUATION: Market: if a competitive market exists, market price = WTP at the margin (use observed prices) Non-market: stated preference (survey) or revealed preference (behavior-based) TWO CATEGORIES OF NON-MARKET METHODS: Stated Preference: ask people directly → Contingent Valuation, Choice Experiments Revealed Preference: infer from behavior → Hedonic Price Method, Travel Cost Method
7. Contingent Valuation (CV) — Design, Biases, NOAA▼
CONTINGENT VALUATION: Survey-based method asking respondents their WTP for a hypothetical change in a public good or environmental quality. CV SURVEY STEPS: 1. Define the valuation problem (what good, what change) 2. Decide how to administer (in person, mail, phone, online) 3. Design the survey: a. Initial questions (establish familiarity with the good) b. Description of the good and the change c. Valuation question with a payment vehicle (how they'd pay) d. Control variables (income, age, education) 4. Analyze the data (logistic regression → WTP estimate) PAYMENT VEHICLE: The mechanism through which payment is made (e.g., higher taxes, entry fees, donations). Must be credible and relevant to the population. Wrong vehicle → protest responses. PROTEST ZEROS: A $0 WTP response that reflects rejection of the survey design, NOT true zero value. Examples: "The government should pay, not me" or "I refuse to put a price on nature." → Identify via follow-up questions. Usually excluded from WTP estimates. CV BIASES: Hypothetical bias: people overstate WTP when no real money changes hands Strategic bias: respondents report high/low WTP to influence policy outcome Starting point (anchoring) bias: bid amount anchors the response Payment vehicle bias: WTP varies by how payment is framed Framing/embedding bias: WTP for wetland ≈ WTP for all environment (scope insensitivity) Interview fatigue: later questions in long surveys get less careful answers General information bias: context/framing of the good affects stated values NOAA BLUE RIBBON PANEL (1993) RECOMMENDATIONS: Use conservative estimates; use WTP (not WTA) Prefer personal (in-person) interviews Use dichotomous choice (yes/no) format — not open-ended Remind respondents of substitute goods and budget constraints Pre-test for scope sensitivity (WTP for more should be > WTP for less) Random sampling; report response rates; ask follow-up for "no" responses
8. CV Estimation — Logistic Regression & WTP Calculation▼
LOGISTIC REGRESSION FOR CV: Binary outcome: y_k = 1 (yes) or 0 (no). Model: p(x) = e^(β₀+β₁x) / (1 + e^(β₀+β₁x)) Estimate β by maximizing log-likelihood: l = Σ_k [y_k · ln(p_k) + (1−y_k) · ln(1−p_k)] REGRESSION FORM: ŷ_k = β̂₀ + β̂₁·Bid_k + β̂₂·X₁ₖ + β̂₃·X₂ₖ + β̂₄·X₃ₖ where ŷ = log-odds of saying "yes" WTP FORMULA: WTP_k = −(β̂₀ + β̂₂·X₁ₖ + β̂₃·X₂ₖ + β̂₄·X₃ₖ) / β̂₁ KEY: Do NOT multiply by Bid_k. The β̂₁ is the coefficient on bid, and you're finding where the predicted log-odds = 0 (50% chance of yes), which gives mean WTP. INTERPRETATION: β̂₁ should be NEGATIVE (higher bid → less likely to say yes) β̂₀ + controls = baseline log-odds at Bid = 0 WTP = the bid at which the respondent is exactly indifferent (50% chance of yes) CHOICE EXPERIMENTS (alternative to CV): Respondents choose between bundles of attributes (e.g., Beach A has cleaner water + higher entrance fee vs. Beach B). Can decompose WTP by individual attribute. Less vulnerable to some CV biases; can capture trade-offs more naturally. Also a stated preference method → also subject to hypothetical bias.
9. Revealed Preference — Hedonic Price & Travel Cost▼
HEDONIC HOUSE PRICE METHOD: People pay more for houses with better environmental quality (lower pollution, proximity to parks, quieter neighborhoods). We can infer WTP for environmental amenities from house price data. Method: 1. Collect data: house prices + structural attributes (beds, baths, sqft) + neighborhood + environmental quality 2. Run regression: price_i = β₀ + β₁·rooms_i + β₂·lot_i + β₃·pollution_i + β₄·school_i + ε_i 3. WTP for 1-unit improvement in environmental variable = |β₃| (the coefficient) Limitations: requires many assumptions about functional form; may not capture option/nonuse values; capitalization may be incomplete; endogeneity. TRAVEL COST METHOD (TCM): People reveal their WTP for a recreational site through how much they're willing to spend to get there (travel + time cost). Method: 1. Survey visitors, record travel cost (distance × cost/mile + time value) 2. Estimate demand: Visits_i = f(TC_i, income, demographics) 3. Consumer surplus = area under demand curve above travel cost 4. PV of site = CS / r (for an infinite stream of annual benefits) Key formulas: CS = ∫₀^{V*} [TC(V) − TC_actual] dV (area between demand and price line) Or: CS = area under demand curve from 0 to V* minus actual expenditure PV = CS / discount rate r (if benefits are perpetual) BENEFITS TRANSFER: Taking WTP estimates from an existing study ("study site") and applying them to a new policy context ("policy site"). Pros: cheap, fast, practical for regulatory decisions. Cons: populations differ, sites differ, time has passed, methods may not match. Best practice: use multiple studies, adjust for income differences, test sensitivity.
10. Value of Statistical Life (VSL)▼
WHAT VSL IS (and is NOT): VSL is the aggregate WTP across a population for a marginal reduction in the risk of death. It is NOT the value of any specific individual's life. Example: 10,000 workers each willing to pay $600 for a safety improvement that reduces annual mortality risk by 1-in-10,000. VSL = $600 / (1/10,000) = $600 × 10,000 = $6,000,000 (Equivalently: 10,000 × $600 = $6M total WTP to prevent 1 expected death) HEDONIC WAGE TECHNIQUE: Workers who accept riskier jobs earn wage premiums — their wage reveals the implicit price of risk. Steps: 1. Collect data on wages, job-specific risk rates, worker characteristics 2. Regression: wage_i = β₀ + β₁·risk_i + β₂·education_i + β₃·experience_i + ... + ε_i (risk measured as annual probability of death per worker) 3. VSL = β̂₁ / (one unit of risk) = β̂₁ × 10,000 (if risk measured per 10,000) MATH FOR EXAMS: Given: WTP per person = X for risk reduction of 1/N VSL = X × N Cost-benefit with VSL: Benefits = VSL × Number of lives saved If Benefits > Costs → implement policy Break-even VSL = Cost / Lives saved CURRENT ESTIMATES: EPA ≈ $10–11M (2024). DOT uses ~$12M. Higher VSL → more policies pass CBA. CONCERNS: Heterogeneity: lower-income workers accept more risk for less → lower implied VSL → using one VSL for everyone raises equity issues Revealed vs. stated: hedonic wage only captures mortality risk; doesn't capture morbidity, quality of life
11. Integrated Assessment Models (IAMs)▼
IAM DEFINITION: Mathematical/data-driven models that link physical/ecological systems to economic systems to estimate costs and benefits of pollution or climate policies. Used because regulators don't know the marginal benefit curve — IAMs estimate it. WHY IAMs? The marginal abatement cost (MAC) is known by firms. The marginal damage (MB) is NOT known — IAMs try to estimate it. TYPES OF POLLUTION DAMAGES (from class): Human health (mortality, morbidity) Environmental damage (ecosystem services, biodiversity) Market sector damage (reduced agricultural yields, lower manufacturing output) Capital depreciation (infrastructure damage from storms, sea-level rise) POINT vs. NONPOINT SOURCE: Point source: discrete, identifiable source (e.g., power plant smokestacks) → top 10,000 US emitters Nonpoint source: diffuse, difficult to attribute (e.g., agricultural runoff, vehicle exhaust) AIR POLLUTION IAM — 5 STEPS (Muller & Mendelsohn 2007): 1. EMISSIONS: Identify point sources. Collect data on location + quantity emitted (EPA data on top emitters). 2. DISPERSION: Atmospheric science. Model how pollution spreads based on wind speed, direction, terrain. 3. EXPOSURE: Map who/what is in harm's way — populations, ecosystems, economic sectors. 4. DOSE-RESPONSE: Toxicology & epidemiology. How does a given change in pollution lead to health/ecosystem impacts? 5. VALUATION: Convert impacts to dollar values using nonmarket valuation (VSL for mortality, hedonic for house values, etc.). KEY FINDING — Urban vs. Rural: ~52% of emissions are urban, ~48% rural. But urban areas generate ~73% of total damages vs. ~27% rural. Why? Exposure — urban areas have higher populations per unit of pollution. TROPICAL CYCLONE IAM EXAMPLE: CO₂ emissions → temperature increase → hurricane intensity/frequency increase → property damage Same 5-step structure applied to storm damages.
12. Climate Change Economics▼
STOCK vs. FLOW POLLUTANTS: Flow pollutant: damages occur when emitted; stops when emissions stop (e.g., SO₂, NOₓ). Efficient abatement: max B(A) − C(A) each period. Stock pollutant: accumulates over time; damages persist long after emissions stop (e.g., CO₂ lasts centuries). Efficient abatement: max PV of future benefits minus PV of costs. CO₂ is a stock pollutant → damages are future damages → discounting matters enormously. SOCIAL COST OF CARBON (SCC): Definition: the present value of the marginal damage from emitting one additional tonne of CO₂ today. Used in regulatory cost-benefit analysis to value emission reductions. Current US government estimate (Biden era): ~$51–$190/tonne depending on discount rate. SCC is an output of an IAM. DISCOUNT RATE DEBATE (most important conceptual issue in climate economics): High r (Nordhaus, ~5%): future damages heavily discounted. SCC ≈ $30–50/tonne. → Less aggressive policy justified today. Low r (Stern, ~1–2%): future damages count nearly as much as present. SCC ≈ $200–300/tonne. → Much more aggressive policy justified. Two sources of disagreement: (1) Empirical: what is the right real interest rate? Market rates vs. social rates differ. (2) Ethical (normative): pure rate of time preference — do we value future generations' welfare equally to present? Most economists say yes (r_pure = 0); markets disagree. PRESENT VALUE MATH: PV = FV / (1+r)^t NPV = Σ_t (B_t − C_t) / (1+r)^t Perpetuity: PV = Annual Flow / r MITIGATION COSTS vs. DAMAGES: Mitigation costs: cost of reducing CO₂ emissions (switching energy sources, efficiency) Damages: economic losses from climate change (heat stress, sea level rise, storms, agriculture) Efficient climate policy: balance marginal mitigation cost = marginal damage (at SCC) CLIMATE AS GLOBAL PUBLIC GOOD: CO₂ is non-rival (everyone affected by same stock) and non-excludable (can't keep countries from benefiting from others' reductions). Each nation has incentive to free-ride on others' efforts. No global enforcement mechanism → tragedy of the commons / prisoner's dilemma. Solutions attempted: Paris Agreement (voluntary, NDCs), EU ETS, carbon border adjustments (CBAMs), technology transfer.
